# 🤝 Руководство по содействию (Contributing Guide)

Привет! Рады, что ты хочешь помочь в развитии **dwarf-sandbox**. Проект полностью открыт для новых идей, оптимизаций и кастомных модификаций.
---

## 📦 Как предложить изменения (Pull Requests)

1. **Сделай Форк**: Сделай ответвление репозитория в свой профиль GitHub.
2. **Создай ветку**: Создай новую ветку для своей фичи (`git checkout -b feature/cool-new-mod`).
3. **Напиши код**: Реализуй механику, придерживаясь чистоты кода и структуры проекта.
4. **Проверь запуск**: Убедись, что игра запускается двойным кликом по `файлу` и не выдает синтаксических ошибок в консоли.
5. **Сделай коммит**: Напиши понятное описание изменений (`git commit -m "Добавил анимацию взмаха кирки"`).
6. **Отправь PR**: Создай Pull Request в ветку `main` нашего основного репозитория.

---

## 🎨 Требования к добавлению новых текстур

Если ты рисуешь новые блоки или инструменты (например, в MS Paint):
* **Формат файлов**: Строго `.png` с маленькими буквами в названии (например, `iron_ore.png`).
* **Разрешение**: Рекомендуемый размер — `32x32` или `40x40` пикселей.
* **Фон**: Желательно рисовать на чисто белом фоне (RGB `255, 255, 255`), так как движок игры автоматически обрезает его на лету и делает полностью прозрачным.

---

## 🛠️ Правила для разработки модов (`/Mods`)

Если ты пишешь дополнение для папки `Mods`:
* Мод не должен напрямую изменять файлы ядра игры (`game.exe`).
* Используй модульный подход, чтобы новые блоки регистрировались через автоматический сканер папки `Mods`.
* Все зависимости для твоего мода должны быть совместимы с библиотеками из `requirements.txt`.

---

## 🐛 Как сообщить об ошибке (Bug Reports)

Если игра вылетает или работает некорректно:
1. Открой раздел **Issues** в репозитории.
2. Прикрепи полный текст ошибки (Traceback) из командной строки Windows.
3. Опиши по шагам, какие действия (клики, клавиши) привели к поломке.
4. Если есть возможность — прикрепи скриншот экрана.
